權限層級

管理者權限

權限概念

通常網站使用者的角色地位中，一般會員與後台管理員主要的不同，在於多少網站中的功能是被授權使用，而最簡單的區分方法，舉例如下 :

  • 一般會員 : 只能控制自己個人的資料。
  • 管理員 : 能控制所有一般會員的資料。

但在較大型的網站中，管理員又可以再細分為不同的地位層級，但因為此系統為單純的NFT管理系統，所以權限設計上可能不會到非常複雜或甚至不做層級上的區分。


前後台概念

根據主要架構此章節的介紹，為了能更加理解此系統架構的設計原因與好處壞處，我們必須額外探討，後台與前台在不同架構下會產生的問題，以及一般設計上的做法。

大致上可以把前後台的架構設計分為兩種，如下 :

  • 伺服器分離 :

    一般來說最保險的做法，是讓運作後台管理系統的伺服器與前台官網伺服器做完全的分離(後台網頁不嵌入在前台網頁之中)，即管理員並非透過前台網頁登入後台管理系統，而是直接訪問後台本身伺服器上的網頁。但這個做法會需要前後台各自的伺服器提供網頁使用，不過好處則是可以預防非管理者進入後台的風險，甚至是讓有心人士需額外花費成本突破後台伺服器。

  • 伺服器合併 :

    不過開發初期時因為考量到成本與資源集中管理，最後還是選擇前後台整合至同一伺服器上運作。

    在前台的地方提供一個管理員帳號(可訪問一般會員不能訪問的網頁元素)，利用此帳號登入後，可以透過一個按鈕連結，連結至後台
